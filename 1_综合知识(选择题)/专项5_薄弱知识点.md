## 一、PV操作

### 1.1 基本规则

- **P(S)**：S=S-1，若S<0则**阻塞等待**
- **V(S)**：S=S+1，若有等待进程则**唤醒一个**

**口诀：P=申请等待，V=释放通知**

### 1.2 生产者-消费者（最基础模型）

一个缓冲区，容量为N。empty=N（空位），full=0（数据），mutex=1（互斥锁）：

```
生产者: P(empty); P(mutex); 写数据; V(mutex); V(full);
消费者: P(full);  P(mutex); 读数据; V(mutex); V(empty);
```

**要点：**
- P的顺序不能反（先P(empty)再P(mutex)，否则死锁）
- V的顺序无所谓
- **当N=1时可以省略mutex**（empty和full已经隐含互斥）

### 1.3 互斥访问（最简单）

两个进程互斥进入临界区，信号量mutex=1：

```
进程: P(mutex); 临界区; V(mutex);
```

### 1.4 前趋同步

P1执行到某点后P2才能开始。信号量S=0：

```
P1: ...执行到某点...; V(S);    // 先完成的V
P2: P(S); ...继续执行...;       // 后开始的P
```

### 1.5 多读者-写者问题

多个读者可同时读，写者写时排斥所有人。

```
读者进程:
  P(mutex); readcount++;
  if(readcount == 1) P(rw);    // 第一个读者锁住写者
  V(mutex);

  读数据;

  P(mutex); readcount--;
  if(readcount == 0) V(rw);    // 最后一个读者解锁
  V(mutex);

写者进程:
  P(rw); 写数据; V(rw);
```

**三个信号量的作用：**
- rw=1：写者之间互斥 + 读者和写者互斥
- mutex=1：保护readcount变量的修改
- readcount=0：记录当前有多少个读者

**为什么第一个读者P(rw)？** 因为如果有写者在写，rw已经被P了，第一个读者也会阻塞，等写者V(rw)后才能读。

### 1.6 做题技巧

看到PV操作的题目，先问自己三个问题：
1. **谁等谁？** → 先完成的V，后开始的P
2. **有没有互斥？** → 共享资源需要mutex
3. **信号量初始值？** → 一般题目会给出

## 二、IP子网划分

### 2.1 核心公式

```
子网数 = 2^n（n=借的主机位数）
每子网主机数 = 2^m - 2（m=剩余主机位数，减2是网络地址和广播地址）
```

### 2.2 常用子网掩码对照表

| CIDR | 子网掩码 | 借位数 | 子网数 | 每子网主机数 |
|------|---------|:------:|:-----:|:----------:|
| /25 | 255.255.255.128 | 1 | 2 | 126 |
| /26 | 255.255.255.192 | 2 | 4 | 62 |
| /27 | 255.255.255.224 | 3 | 8 | 30 |
| /28 | 255.255.255.240 | 4 | 16 | 14 |
| /29 | 255.255.255.248 | 5 | 32 | 6 |
| /30 | 255.255.255.252 | 6 | 64 | 2 |

### 2.3 求网络地址

**网络地址 = IP地址 AND 子网掩码**

例：IP 192.168.10.130/26
```
130 = 10000010
/26掩码最后字节 = 11000000 = 192
AND运算：10000010 AND 11000000 = 10000000 = 128
网络地址 = 192.168.10.128
```

### 2.4 做题步骤

1. 看需要几个子网 → 确定借几位
2. 看每子网要多少主机 → 确定留几位
3. 查表得出子网掩码

## 三、等级保护2.0

### 3.1 五个级别

| 级别 | 名称 | 备案 | 测评 | 适用 |
|:---:|------|:---:|:---:|------|
| 一级 | 自主保护级 | 不需 | 不需 | 小型系统 |
| 二级 | 指导保护级 | **需备案** | **不需强制测评** | 一般系统 |
| 三级 | 监督保护级 | 需备案 | **每年至少1次** | 重要系统（银行、电商） |
| 四级 | 强制保护级 | 需备案 | **每半年至少1次** | 极重要系统（国防） |
| 五级 | 专控保护级 | 需备案 | 更频繁 | 国家安全 |

### 3.2 考试高频陷阱

- ❌ "第二级以上需要强制测评" → 错！**第三级以上**才需要
- ❌ "等级保护共分三级" → 错！**五级**
- ✅ "第一级不需备案" → 对
- ✅ "第四级要求追踪溯源" → 对

## 四、关键路径

### 4.1 基本概念

- **关键路径**：从起点到终点**最长**的路径（决定项目最短完成时间）
- **关键活动**：总时差=0的活动（delay了就delay整个项目）

### 4.2 总时差计算

```
总时差 = 最迟开始时间LS - 最早开始时间ES
```

**总时差=0的活动就是关键活动**，关键活动连起来就是关键路径。

### 4.3 做题方法

**方法一：列举所有路径（适合小项目）**

画出所有从起点到终点的路径，计算每条路径的总工期，最长的就是关键路径。

例：
```
路径1: A→C→E→F = 5+4+3+2 = 14天
路径2: B→D→E→F = 3+6+3+2 = 14天
```
关键路径长度=14天。

**方法二：用ES和LS（适合给了表格的题）**

总时差=LS-ES=0的活动找出来，连成路径。

### 4.4 易错点

- 关键路径**不一定只有一条**
- 关键路径上的活动**不能delay**（delay一天整个项目晚一天）
- 非关键活动的总时差=可以拖延的天数而不影响总工期

## 五、OPT页面置换

### 5.1 规则

**淘汰将来最久不会被使用的页面**（理论最优，实际无法实现，因为无法预知未来）

### 5.2 手算方法

1. 正常装入前几页（页框空时直接装入）
2. 缺页时，**向后看**剩余的访问序列
3. 哪个页面将来最晚出现（或不再出现），就淘汰哪个
4. 如果多个页面将来都不再出现，淘汰任意一个

### 5.3 例题

3个页框，序列：3, 2, 1, 3, 2, 4, 3, 2, 1, 4

第6步访问4，页框{1,2,3}，看将来[3,2,1,4]：
- 3→马上要用（位置7）→留
- 2→位置8用到→留
- 1→位置9才用→最远→**淘汰1**
