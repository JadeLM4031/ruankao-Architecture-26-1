## 一、流水线计算

### 1.1 三个核心公式

**时钟周期**（所有阶段必须等最慢的那个）：
> Δt = max(各段时间)

**总执行时间**：
> T = (级数k + 指令数n - 1) × Δt

为什么？第1条指令要走完k个阶段，但从第2条开始，每个时钟周期就能流出一条（因为各阶段并行了）。

**吞吐率**：
> TP = n / T

**加速比**：
> S = 不用流水线的时间 / 用流水线的时间

### 1.2 计算示例

4级流水线，各段2ns/3ns/2ns/2ns，100条指令：
- Δt = max(2,3,2,2) = **3ns**
- T = (4+100-1)×3 = 103×3 = **309ns**
- TP = 100/309 ns⁻¹

如果不加流水线：每条指令要2+3+2+2=9ns，100条=900ns
- 加速比 S = 900/309 ≈ **2.91**

### 1.3 气泡（stall）

数据冒险等原因需要在流水线中插入等待周期：
- 每条指令（除第一条外）插入m个气泡
- 实际时间 = 理想流水线时间 + m×(n-1)×Δt

## 二、LRU页面置换

### 2.1 核心规则

**LRU = 淘汰最近最久没用的页面**

手算方法：每一步维护一个列表，**左边=最近使用的，右边=最久没用的**

- **命中**：把该页移到最左边
- **缺页**：淘汰最右边的，新页加到最左边

### 2.2 计算示例

3个页框，访问序列：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

```
步骤  访问  页框（近→久）    命中/缺页
1     1    {1}              缺页
2     2    {2,1}            缺页
3     3    {3,2,1}          缺页
4     4    {4,3,2}          缺页（淘汰1）
5     1    {1,4,3}          缺页（淘汰2）
6     2    {2,1,4}          缺页（淘汰3）
7     5    {5,2,1}          缺页（淘汰4）
8     1    {1,5,2}          命中（1移到最左）
9     2    {2,1,5}          命中（2移到最左）
10    3    {3,2,1}          缺页（淘汰5）
11    4    {4,3,2}          缺页（淘汰1）
12    5    {5,4,3}          缺页（淘汰2）
```

**共10次缺页**

### 2.3 Belady异常

- **定义**：增加页框数反而导致缺页率上升
- **只有FIFO可能出现**
- LRU和OPT是**栈式算法**，不会出现Belady异常
- 栈式算法：n个页框中的页面 ⊆ (n+1)个页框中的页面

### 2.4 三种算法对比

| 算法 | 淘汰策略 | Belady异常 |
|------|---------|-----------|
| OPT | 淘汰将来最久不用的 | 不会 |
| LRU | 淘汰过去最近最久未用的 | 不会 |
| FIFO | 淘汰最早进入的 | **可能** |

## 三、PV操作

### 3.1 基本概念

- **P(S)**：申请资源，S-1；若S<0则阻塞等待
- **V(S)**：释放资源，S+1；若有等待进程则唤醒一个

**口诀：P=等（申请），V=通知（释放）**

### 3.2 生产者-消费者问题

一个缓冲区，P1写、P2读。empty=1（空位），full=0（数据）：

```
P1（生产者）：P(empty) → 写数据 → V(full)
P2（消费者）：P(full)  → 读数据 → V(empty)
```

**口诀：生产者P空V满，消费者P满V空**

### 3.3 多读者问题

P1写，P2和P3读，要求P1写完后P2/P3都能读，P2/P3都读完后P1才能再写：

```
P1: V(s2); V(s3); P(finish); P(finish);   // 通知两人可读，等两人都读完
P2: P(s2); 读; V(finish);                  // 等P1通知，读完通知P1
P3: P(s3); 读; V(finish);                  // 等P1通知，读完通知P1
```

**关键点：**
- P1需要V两个不同信号量（s2给P2，s3给P3），不能用同一个
- P1需要P(finish)两次，确保P2和P3**都**读完

### 3.4 哲学家进餐问题（防死锁）

5个哲学家围坐圆桌，每两人间1根筷子。防死锁方案：

**奇数号先拿左后拿右，偶数号先拿右后拿左** → 破坏循环等待

## 四、磁盘调度

### 4.1 SCAN（电梯算法）

沿当前方向扫描到端，然后反向继续扫描。

**例：** 当前53号柱面，方向减小，请求：98, 37, 122, 14, 124, 65, 67

```
减小方向：37 → 14（到头）
反向增大方向：65 → 67 → 98 → 122 → 124
```

完整序列：**37, 14, 65, 67, 98, 122, 124**

### 4.2 CSCAN（循环扫描）

只沿一个方向扫描，到头后直接跳回起始端继续同方向。

```
增大方向：65 → 67 → 98 → 122 → 124（到头）
跳回0端继续增大：14 → 37
```

### 4.3 四种调度算法对比

| 算法 | 特点 |
|------|------|
| FCFS | 按请求到达顺序服务 |
| SSTF | 优先服务最近的请求（可能饥饿） |
| SCAN | 电梯算法，到头反向 |
| CSCAN | 到头跳回起点，只单向扫描 |

## 五、Cache映射

### 5.1 三种映射方式

| 方式 | 规则 | 优点 | 缺点 |
|------|------|------|------|
| 直接映射 | 块号 % 行数 = 固定行 | 速度快 | 冲突多 |
| 全相联映射 | 任意块放任意行 | 冲突最少 | 硬件复杂 |
| N路组相联 | 先分组，组内N路任选 | 平衡 | — |

### 5.2 计算公式

```
Cache总行数 = Cache容量 / 块大小
组数（N路组相联）= 总行数 / N
```

### 5.3 计算示例

64KB Cache，64B块，4路组相联：
- 总行数 = 64KB / 64B = **1024行**
- 组数 = 1024 / 4 = **256组**

32KB Cache，64B块，直接映射：
- 行数 = 32KB / 64B = **512行**（直接映射=1路组相联，组数=行数）

16KB Cache，32B块，8路组相联：
- 总行数 = 16KB / 32B = **512行**
- 组数 = 512 / 8 = **64组**

## 六、死锁资源计算

### 6.1 公式

> m ≥ n×(X-1) + 1

- m = 系统资源总数
- n = 进程数
- X = 每个进程最多需要的资源数

**含义：** 最坏情况每个进程各拿(X-1)个（差1个就能完成），再留至少1个给某个进程先完成。

### 6.2 计算示例

12个资源，4个进程，求X最大值：
```
4(X-1)+1 ≤ 12
4X-4+1 ≤ 12
4X ≤ 15
X ≤ 3.75 → X最大=3
```

验证X=3：4个进程各持2个=8个，剩4个，至少1个进程可拿到第3个→完成释放→不会死锁。
验证X=4：4个进程各持3个=12个，剩0个→全部等待→死锁！

## 七、RISC vs CISC

| 特征 | RISC | CISC |
|------|------|------|
| 指令数量 | 少 | 多 |
| 指令长度 | **固定** | **可变** |
| 寻址方式 | **简单**（少） | **丰富**（多） |
| 寄存器数量 | **多**（>100） | 少 |
| 控制器 | **硬布线** | **微程序** |
| 流水线 | **适合**（大多1周期） | 不适合 |
| 代表 | ARM、MIPS | x86 |

**记忆：RISC精简（少指令少寻址多寄存器），CISC复杂（多指令多寻址少寄存器）**

## 八、寻址方式

| 方式 | 有效地址 | 类比 |
|------|---------|------|
| 立即寻址 | 操作数就在指令里 | 直接给你钱 |
| 直接寻址 | 指令中给出操作数地址 | 给你门牌号 |
| 寄存器寻址 | 操作数在寄存器里 | 钱在你口袋 |
| 寄存器**间接**寻址 | 寄存器里存的是操作数地址 | 口袋里有张纸条写着门牌号 |
| 基址寻址 | 寄存器内容 + 偏移量 | 门牌号+楼栋偏移 |
| 变址寻址 | 变址寄存器 + 指令中地址 | 数组遍历（基址+下标） |

**考试高频：** 寄存器间接寻址 → EA = 寄存器中的值（不是寄存器本身，不是寄存器里的内存内容）

## 九、分页地址转换

### 公式
```
页号 = 逻辑地址 / 页面大小（整除）
页内偏移 = 逻辑地址 % 页面大小
物理地址 = 页框号 × 页面大小 + 页内偏移
```

### 示例

页面4KB，页表：0→3, 1→5, 2→8, 3→2。逻辑地址0x2A1F：
```
4KB = 0x1000
页号 = 0x2A1F / 0x1000 = 2
偏移 = 0x2A1F % 0x1000 = 0xA1F
查表第2页→页框8
物理地址 = 8 × 0x1000 + 0xA1F = 0x8A1F
```

### 十六进制快速法

页面大小为4KB=0x1000时：
- 页号 = 高位部分（去掉最后3位十六进制）
- 偏移 = 低3位十六进制

例：0x2A1F → 页号=0x2，偏移=0xA1F
