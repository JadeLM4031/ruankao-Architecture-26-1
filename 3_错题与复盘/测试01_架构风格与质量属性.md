## 题目一：架构风格识别（选择题）✅

**题目：** 某企业需要开发一个语音识别系统，系统包含多个独立的"知识源"模块（如声学分析模块、语言模型模块、语义理解模块），这些模块通过一个共享的数据结构协同工作：每个模块从共享区域读取当前状态，将自己的处理结果写回共享区域，由一个控制器决定下一步激活哪个模块。该系统最适合采用哪种架构风格？

A. 管道-过滤器风格（数据流风格）
B. 层次结构风格（调用/返回风格）
**C. 黑板系统风格（仓库风格）** ← 正确答案
D. 事件驱动风格（独立构件风格）

**我的答案：** C ✅

**解析：**
- "多个知识源" → 黑板系统的知识源构件
- "共享的数据结构" → 黑板（共享数据区）
- "控制器决定激活哪个模块" → 黑板系统的控制器
- 三大构件完全对应：黑板 + 知识源 + 控制器 → 黑板系统

**易错辨析：**
- 不选A：没有逐步变换的数据流，而是多模块协同
- 不选D：通过共享数据交互而非事件触发，黑板是"数据驱动"

## 题目二：质量属性分析（简答题）✅

**题目：** 某在线交易系统采用明文传输用户数据，为提升安全性决定对所有通信数据使用AES-256加密。实施后安全性显著提升，但响应时间从50ms增加到180ms。指出一个敏感点和一个权衡点。

**我的答案：**
- 敏感点：使用AES加密，提升安全性。该决策影响了安全性这一个质量属性。
- 权衡点：安全性提升但响应时间变长，性能降低。该决策影响了两个质量属性，做出了权衡折中。

**评分：** 满分 ✅

**参考满分答案：**
- 敏感点：通信数据的加密方式是**安全性**的敏感点，因为加密强度直接影响系统的安全等级。
- 权衡点：AES-256加密是**安全性与性能**之间的权衡点。提高安全性（明文→AES-256）但降低了性能（50ms→180ms，下降约260%）。

**答题提升建议：**
1. 敏感点用"XX是YY质量属性的敏感点"的标准句式
2. 权衡点点明"哪两个属性矛盾"并补充量化数据
3. 考试时多写一句不扣分，写少了可能丢分
