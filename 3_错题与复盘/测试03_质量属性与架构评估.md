## 错题1：第4题（选择题）— 风险 vs 权衡点

**题目：** 双机热备方案，主备切换时约30秒服务中断。这一发现属于？

**我的答案：** D（权衡点） ❌
**正确答案：** B（风险）

**错因分析：**
- 风险 = 架构决策中已经暴露的问题/隐患
- 权衡点 = 同时影响多个矛盾质量属性的决策
- "30秒中断"是已暴露的隐患，不是"为了A牺牲B"的权衡描述
- 题目只涉及可用性一个维度，没有提矛盾属性

**记忆口诀：**
- 看到"问题/隐患/可能导致" → 风险
- 看到"为了A但牺牲B/两个属性矛盾" → 权衡点
- 看到"只影响一个属性的变化幅度" → 敏感点

## 半对题：第6题案例 问题2 — 权衡点选取

**我的答案：** API网关统一鉴权 → 安全性↑ 性能↓
**参考答案：** 单体拆分为微服务 → 可修改性↑ 性能/可靠性↓

**问题：** 抓小放大。网关鉴权是配套措施，"拆分为微服务"才是最大的架构决策。
**教训：** 案例题找权衡点时，优先选择**题目中描述最重大的架构变更**。
