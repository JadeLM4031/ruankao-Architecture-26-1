# 系统架构设计师 - 知识点模块学习路线

> 只覆盖综合知识（选择题）+ 案例分析，论文暂不涉及
> 按知识点依赖关系排列，非按日期
> 每个模块 = 考点讲解 + 真题仿真练习 + 错题收录

---

## 学习顺序总览

```
模块1 软件架构风格（最高频，案例+选择双杀）
  │
模块2 质量属性与架构评估（案例必答题核心）
  │
模块3 设计模式（选择+案例高频）
  │
模块4 系统开发方法与需求工程
  │
模块5 数据库系统（含计算题）
  │
模块6 系统可靠性与性能评价（含公式计算）
  │
模块7 UML建模（选择高频，案例可能考）
  │
模块8 计算机组成与操作系统
  │
模块9 计算机网络
  │
模块10 信息安全（含零信任ZTA）
  │
模块11 新增考点（云原生/大数据Lambda Kappa/ArchiMate/AI）
  │
模块12 数学与经济管理 + 知识产权 + 项目管理
```

> **原则：** 模块1-3是"三杀"核心（选择+案例+论文都能考），必须最熟。模块4-6是高分章节。模块7-11是稳定得分区。模块12是捡分区。
>
> **每个模块的学习流程：**
> 1. 我给你讲考点（精简、只讲考什么、怎么考、陷阱在哪）
> 2. 出仿真题（题型和难度贴近真题）
> 3. 你作答
> 4. 我评分+解析，错题收录到 `3_错题与复盘/`
> 5. 进入下一个模块
